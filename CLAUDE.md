Config-driven zsh generation + Rust CLI utilities. Replaces shell scripting
in blackmatter-shell with a single binary that reads YAML and emits zsh.

## Architecture

```
~/.config/blx/blx.yaml  →  blx init zsh  →  eval "$(blx init zsh)"
       (YAML)                 (Rust)              (zsh eval)
```

The core model: **YAML config → Rust codegen → zsh eval**. No shell scripting
in the configuration layer. All shell output is generated by Rust with proper
quoting and escaping.

## Key Files

| File | Purpose |
|---|---|
| `src/config.rs` | 18 config types with serde defaults, YAML parse/dump |

| `src/main.rs` | Entry point |
| `flake.nix` | Nix build via substrate `rust-tool-release-flake.nix` |

## Config

Config file: `~/.config/blx/blx.yaml` (override via `BLX_CONFIG` env var).

All fields have `#[serde(default)]` — partial YAML works. Only set what you
want to override; omitted fields use built-in defaults.

### Config Sections

- **shell** — editor, visual, pager, less_opts, bat_theme, manpager, funcnest, keytimeout
- **history** — file, size, save_size, 9 history options (extended, share, etc.)
- **options** — 22 zsh options (auto_cd, extended_glob, etc.)
- **environment** — extra `export KEY='value'` pairs
- **aliases** — alias map (fully replaces defaults when set)
- **tools.eza** — ls wrapper, icons, group_directories_first
- **tools.skim** — algo, height, layout, colors (Nord), ctrl_t/alt_c/ctrl_f widgets
- **completion** — case_insensitive, matchers, menu_select, cache, rehash, colors
- **vim_mode** — enabled, cursor_shape, system_clipboard, edit_command_line
- **plugins** — direnv, zoxide, atuin, fzf_tab, starship, syntax_highlighting
- **extra_zsh** — raw zsh lines appended at the end (escape hatch)

## Zsh Generation Order

The generated zsh follows a strict ordering:

1. Options (setopt/unsetopt)
2. History (HISTFILE, HISTSIZE, history options)
3. Environment (EDITOR, VISUAL, PAGER, user env vars)
4. Completion (compinit, matchers, zstyles)
5. Vim mode (bindkey -v, cursor shape, clipboard)
6. Aliases (user aliases + platform-specific)
7. Eza wrapper (ls() function + aliases)
8. Skim (SKIM_DEFAULT_OPTIONS, widgets, FZF bridge)
9. Plugins (direnv → zoxide → atuin → fzf-tab → starship → syntax-highlighting)
10. Extra zsh lines

Plugin order matters: direnv before starship (hooks), syntax-highlighting last.

## Shell Quoting

All user-controllable values embedded in single-quoted zsh contexts use
`shell_quote()` which replaces `'` with `'\''`. This prevents shell injection
from config values.

## Testing

84 unit tests in `config.rs` (14) and `init.rs` (70):

```
cargo test                          # Run all
cargo test config::tests            # Config parsing only
cargo test init::tests              # Zsh generation only
cargo test shell_quote              # Quoting safety
cargo test injection                # Shell injection tests
```

Test categories:
- Config: YAML roundtrip, partial merge, invalid input, type errors
- Generation: each section independently, feature toggles, plugin ordering
- Safety: shell quoting edge cases, injection vectors
- Roundtrip: YAML → config → zsh determinism

## Integration

blx is installed via the nix repo overlay chain:

```
pleme-io/nix/flake.nix (blx input)
  → nix/overlays/blx.nix (pkgs.blx overlay)
    → blackmatter-shell/module/default.nix (conditionally installs pkgs.blx)
```

After rebuild, `blx` is on `$PATH`. Add to `.zshrc`:
```zsh
eval "$(blx init zsh)"
```
